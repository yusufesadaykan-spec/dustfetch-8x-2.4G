## 🐧 Arch Linux Boot Requirements (How It Works)

To make this custom hardware boot **Arch Linux (RISC-V)**, the following stack is implemented within the Verilog layers and the boot memory:

1. **Privileged Modes:** Cores support `Machine Mode (M)`, `Supervisor Mode (S)`, and `User Mode (U)`.
2. **Memory Management Unit (MMU):** Features an **Sv39** hardware Page Table Walker to allow virtual-to-physical memory translation required by the Linux Kernel.
3. **Interrupts & Timers:** Integrated **PLIC** (Platform-Level Interrupt Controller) and **CLINT** (Core Local Interruptor) for system scheduling.
4. **Storage:** Standard SPI/SDIO MicroSD controller to read the Arch Linux rootfs.
